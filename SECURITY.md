# Security Policy

## Reporting a vulnerability

Please do not disclose security issues publicly in GitHub issues, discussions, or
pull requests.

Use GitHub's private vulnerability reporting for this repository when it is
available. If private reporting is not enabled, open a minimal public issue that
does not include exploit details and asks for a private reporting path.

## Supported versions

Security fixes are provided on a best-effort basis for the current `main` branch
and the most recent public release tag.

## Security notes

Operational security details, hardening defaults, and release-readiness helpers
are documented in [docs/security.md](docs/security.md).
